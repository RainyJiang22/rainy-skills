# 状态管理、路由与动画

## 状态管理（表现层）

1. **方案统一**：全项目一种主方案（Riverpod、Bloc、Provider 等），与 Flutter 官方对齐；同模块不混用多种写法。
2. **作用域清晰**：状态与页面/模块/全局对应；避免全局单例持有过大状态；大状态懒加载与释放。
3. **依赖注入**：通过 DI 或 Riverpod/GetIt 显式注入；避免深层 Get.find() 或散落单例。
4. **异步与副作用**：集中处理（async notifier、Cubit）；Loading/Data/Error 统一表示，不在 UI 散落 try/catch。

## 路由与导航（表现层）

1. **声明式路由**：优先 GoRouter 或 Navigator 2.0；路由表集中，支持深链、参数、守卫。
2. **命名与参数**：路由名、参数 key 用常量或枚举；传参类型明确，避免强转。
3. **与状态解耦**：路由不依赖全局单例；通过路由参数或 DI 获取数据。

## 动画

1. **优先官方 API**：AnimationController、Tween、Curve、ImplicitlyAnimatedWidget；Lottie 等第三方需统一封装。
2. **性能**：60fps；不在动画回调里重计算或同步 I/O；列表内动画配合懒加载与复用。
3. **可配置**：时长、曲线、延迟用常量或主题；支持「减少动效」等无障碍设置。
4. **与状态解耦**：动画状态与业务状态分离；业务不依赖动画帧或实现细节。
